CSS属性书写顺序：
    1.布局定位属性
        display/position/float/clear/visibility/overflow
        display第一个写，关系到模式
    2.自身属性
        width/height/margin/padding/border/background
    3.文本属性
        color/font/text-decoration/text-align/vertical-align/white-space/break-word
    4.其他属性
        content/cursor/border-radius/box-shadow/text-shadow/background:linear-gradient
示例：
.jdc {
    display: block; 第一档
    position: relative;
    float: left;

    width: 100px; 第二档
    height: 100px;
    margin: 0 10px;
    padding: 20px 0;

    font-family: Arial, "Helvetica Neue", Helvetica, sans-serif;
    color: #333; 第三档

    background: rgba(0,0,0,.5); 第四档
    border-radius: 10px;
}

页面布局整体思路：
1.先确认页面的版心（可视区），可以测量可知
2.分析页面中的行模块，以及每个行模块中的列模块，符合页面布局第一准则
3.一行中的列模块经常浮动布局，先确定每个列的大小，再确定列的位置，符合页面布局第二准则
4.制作html结构，遵循先有结构，后有样式的原则，结构最重要
5.理清布局结构，再写代码

学成在线案例：
1）确定版心，版心的像素是1200像素，每个版心要居中对齐，所以可以定义版心为公共类
.w {
    width: 1200px;
    margin: auto;
}
2）第一部分：head的制作
分为logo + （首页+课程+职业规划）+（输入关键字）+（个人中心+小图+qq-leishui）共4个列
整行：header 1200*42的盒子水平居中对齐，上下给1个margin
列1：logo图片
列2：nav导航栏
列3：search搜索框
列4：user个人信息
这4个列必须是浮动

关于列2的导航栏，实际开发中不会直接用链接a，而是使用li包含链接a的做法
因为li+a的语义更清晰。有条理的列表型内容
如果直接使用a，搜索引擎容易识别为有堆砌关键字嫌疑，故意堆砌关键字容易有被搜索引擎降权的风险，从而影响网站排名

现在导航栏的3行文字怎么放在一行，让li浮动还是让链接a浮动？
答案是要给li加浮动，因为li是块级元素，需要1行独占显示，加了浮动就可以同时成为一行
而a是包含于li的，跟随父元素
导航栏可以不给宽度，将来可以继续添加其余文字
因为导航栏文字不一样多，所以给链接a左右padding撑开盒子而不是指定宽度

搜索框有个大盒子，有高度和宽度，内部再包含输入文字框和一个按钮图片

3）第二部分：banner的制作
最大的盒子banner，不给宽度，给高度，给一个蓝色背景，这个是跟随w的，横跨浏览器
然后版心盒子，水平居中对齐，也就是中间的部分
版心内左边是侧边栏，右边是课程表

banner下边是一行文字，需要一个大盒子，水平居中有盒子阴影
然后左侧的"精品推荐"文字标题h3左侧浮动
中间的"Jquery、Spark、MySQL、JavaWeb"方链接也是左侧浮动，文字距离可以控制链接的左右外边距（行内元素只有左右内外边距）
右侧的"修改兴趣"文字右浮动

4) 精品推荐的小模块与大模块
小模块就在div内设置h3和ul>li>a即可

大模块有个最大的盒子，box版心水平居中对齐
然后大模块内分为2部分，上部分box-hd和下部分box-bd
上部分相对较窄，左侧是标题h3左浮动，右侧是链接右浮动
下部分里面是无序列表，共10个小li构成，每个li要注意好外边距的问题
注：每个li的宽度加上间隙宽度之和，再乘5作为ul的宽度，那么每个ui就可以装5个li不会换行

5）底部的制作
整个的大盒子需要给定高度，背景色白色
之后盒子内部有个盒子，是版心，水平居中
版心盒子内部又有2个盒子，左边是版权copyright，右边是链接组links
链接组使用自定义列表实现，也就是dt和dd
